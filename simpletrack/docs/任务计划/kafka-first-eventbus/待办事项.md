## 任务细化

### KAFKA-HARDEN-001：真实 Kafka broker 集成验证

- 背景：v1 已有 gated integration test 入口，但本次收口没有跑真实 Kafka broker 验证。
- 执行入口：优先使用 `src/analytics-core/docker-compose.yml` 中的本地单节点 Kafka。
- 必须覆盖：正常 publish/consume/commit；handler 随机 sleep 下无提前提交；consumer kill/restart 后未提交消息可重放。
- 完成记录：回填 Kafka 镜像版本、执行命令、通过的测试名、失败坑位和对应处理。

### KAFKA-HARDEN-002：Producer 可靠性默认值评审与落地

- 背景：当前 Kafka provider 已能发布 JSON `EventEnvelope`，但生产默认可靠性策略还未拍板。
- 决策点：是否把 Sarama `Producer.RequiredAcks = WaitForAll` 作为生产默认；是否启用 idempotent producer；retry 和 flush 默认值是否按生产路径收紧。
- 验收重点：配置缺失时的错误信息明确；可靠性默认值有单测保护；如启用 idempotence，必须处理 Sarama 对 `Net.MaxOpenRequests` 等相关约束。

### KAFKA-HARDEN-003：Consumer rebalance 与 ordered commit 集成压测

- 背景：ordered committer 已覆盖乱序完成，但 rebalance / claim 关闭场景仍是生产风险点。
- 必须覆盖：`ConsumeClaim` 结束时 in-flight message 的处理策略；partition committer stop 后是否还允许提交；session 生命周期结束前的 flush/abort 边界。
- 验收重点：不能因为后面的 offset 完成较早而跳过前面的失败消息；不能在 DLQ 写失败时提交原 offset。

### KAFKA-HARDEN-004：DLQ / retry / consumer lag / pause-resume 可观测性

- 背景：v1 已有 retry、DLQ 和 protector 基础逻辑，但生产排障需要可观察信号。
- 候选指标：consume count、handler success/failure、retry count、DLQ write success/failure、partition lag、in-flight count、pause/resume state、ordered commit next offset。
- 验收重点：指标或日志不泄漏事件敏感 payload；service 侧能定位 topic、partition、offset、consumer group 和 DLQ topic。

### KAFKA-HARDEN-005：SASL/TLS 与 broker ACL 配置矩阵

- 背景：第一版明确不做 SASL/TLS 完整配置矩阵，但生产接入前需要补齐。
- 配置范围：brokers、client id、consumer group、topic、DLQ topic、SASL mechanism、username/password、TLS enable、CA/cert/key 路径或内容、ACL 需求说明。
- 验收重点：不把密钥写入文档样例；配置解析和缺失项报错可测试；本地无认证 broker 时允许只做配置层测试。

### KAFKA-HARDEN-006：KafkaBus 性能基线

- 背景：已有 Redis Stream 基线，KafkaBus 还缺同口径 benchmark。
- 建议口径：publish 单事件、publish batch 候选、consume + handler no-op + ordered commit、失败 retry、DLQ 写入。
- 验收重点：明确 benchmark 不等同 SLA；记录本机 CPU、Kafka 镜像、partition 数、message size、parallelism 和 commit batch 策略。

### KAFKA-HARDEN-007：生产 runbook 与故障演练文档

- 背景：代码能力落地后，需要 operator-facing 文档让生产接入不靠口头记忆。
- 必须包含：topic 创建建议、partition key 选择、consumer group 命名、env 样例、DLQ replay/补偿、consumer group reset、常见错误、回滚到 Redis Stream 的限制说明。
- 验收重点：文档只写可执行步骤和真实配置名，不写空泛架构宣传。
