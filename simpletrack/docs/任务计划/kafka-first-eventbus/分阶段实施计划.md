# Kafka-First EventBus 分阶段实施计划

> 状态：实现已完成，阶段评审与正式文档同步中
> 最近更新：2026-05-17

## 总目标

把 `analytics-core` 的 EventBus 改造成 Kafka 优先模型：

- Kafka 是生产主 provider，第一版使用 `github.com/IBM/sarama`。
- Redis Stream 保留为本地、小流量、测试 provider。
- 公共 `eventbus` 契约不泄漏 Sarama session、claim、MarkMessage、MarkOffset 等实现细节。
- 复用或轻改 `xwl_bi` 已验证的 ordered commit、completion gate、consumption protector、worker pool 思路。
- handler 只返回 `error`，provider 内部负责 retry、DLQ、ordered commit。

## 阶段 3：Kafka provider 核心实现

状态：已完成

交付物：

- `eventbus/kafka` 使用 Sarama producer 发布 JSON `contracts.EventEnvelope`。
- 内部 `sessionAdapter` 封装 Sarama consumer group callback，不向公共契约泄漏 Sarama。
- `partitionOrderedCommitter` 支持同 partition 内乱序完成、连续区间顺序提交。
- `MessageCompletionGate` 支持一条消息多个异步任务完成后才进入 ordered commit。
- 固定容量 `DynamicWorkerPool` 承接 handler 并发。
- 轻量 `consumptionProtector` 保留 pending/gate/worker pool/persistence error 信号。
- 最小 DLQ：超过 `MaxAttempts` 后写入 `analytics.events.dead`，DLQ 成功后才完成原 offset。

验收：

- Kafka provider 单测覆盖 publish、session adapter 隔离、ordered commit、completion gate、retry、DLQ 成功提交、DLQ 失败不提交。
- 集成测试默认跳过，设置本地 Kafka env 后可运行 publish -> consume -> commit。
  - 当前已新增 gated 集成测试入口：`ANALYTICS_CORE_KAFKA_INTEGRATION=1 go test ./eventbus/kafka -run TestKafkaIntegrationPublishConsume -v`。

阶段评审：

- 阶段完成后运行 `code-review`。
- 若实现出现重复或样板化，再运行 `code-simplifier` 与 `ai-slop-cleaner`，范围限定为本阶段改动文件。
