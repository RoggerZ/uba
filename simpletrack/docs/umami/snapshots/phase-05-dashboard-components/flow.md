# Phase 05 Flow

1. 起点状态：`Add board` 弹窗已切到 `Website` 类型
2. 用户动作：选择具体 website，并完成 board 基本配置
3. 页面响应：进入 `Design Board` 页面
4. 截图编号：`P05-C01` `P05-C02` `P05-C03` `P05-C04`
5. 用户动作：点击布局级 `+` 按钮
6. 页面响应：出现 `Select component` 弹窗，预览区初始为空
7. 截图编号：`P05-C05`
8. 用户动作：依次选择组件
9. 页面响应：预览区切换为不同组件预览
10. 截图编号：`P05-C06` `P05-C07` `P05-C08` `P05-C09` `P05-C10` `P05-C11` `P05-C12`
11. 用户动作：保存 `Website chart`
12. 页面响应：组件真正落板到设计器
13. 截图编号：`P05-C13`
14. 用户动作：打开 `Filter`，查看 `Fields`、`Match` 下拉、`Segments` tab
15. 页面响应：字段列表、`All / Any` 下拉、Segments 空态都可见
16. 截图编号：`P05-C14` `P05-C15` `P05-C16`
17. 对产品设计的启发：组件系统至少要提供“组件列表 + 预览 + 落板”三段式证据链
