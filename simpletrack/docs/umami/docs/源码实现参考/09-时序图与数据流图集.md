# 09-时序图与数据流图集

## 结论

本章把前面章节的数据点和处理动作收束成图，便于在 SimpleTrack P1 评审、研发拆解和 `analytics-core` 接口设计时直接引用。

## 图 1：端到端数据流

```mermaid
flowchart LR
    A["User opens tracked page"] --> B["tracker builds payload"]
    B --> C["POST /api/send"]
    C --> D["validate + enrich + derive session"]
    D --> E["saveEvent / saveSessionData"]
    E --> F["website_event"]
    E --> G["event_data"]
    E --> H["session_data"]
    F --> I["Realtime query"]
    F --> J["Events query"]
    G --> J
    H --> I
```

## 图 2：Collect 服务端时序

```mermaid
sequenceDiagram
    participant Browser
    participant Tracker
    participant SendAPI as send route
    participant Detect
    participant Writer
    participant Storage

    Browser->>Tracker: pageview / click / identify
    Tracker->>SendAPI: {type, payload}
    SendAPI->>SendAPI: Zod schema validation
    SendAPI->>Detect: get client info
    Detect-->>SendAPI: ip, ua, geo, device
    SendAPI->>SendAPI: bot and blocked IP checks
    SendAPI->>SendAPI: sessionId / visitId
    SendAPI->>Writer: saveEvent or saveSessionData
    Writer->>Storage: insert event rows
    SendAPI-->>Tracker: cache token
```

## 图 3：写入分支

```mermaid
flowchart TD
    A["saveEvent(args)"] --> B{"CLICKHOUSE_URL?"}
    B -- no --> C["Prisma websiteEvent.create"]
    B -- yes --> D{"kafka.enabled?"}
    D -- yes --> E["Kafka message: event"]
    D -- no --> F["ClickHouse insert website_event"]
    A --> G{"eventData exists?"}
    G -- yes --> H["flattenJSON"]
    H --> I["event_data rows"]
    I --> J{"backend"}
    J -- Prisma --> K["eventData.createMany"]
    J -- ClickHouse/Kafka --> L["insert/send event_data"]
```

## 图 4：Realtime 查询

```mermaid
flowchart TD
    A["useRealtimeQuery"] --> B["GET /api/realtime/:websiteId every interval"]

    D --> E["getRealtimeActivity"]
    D --> F["getPageviewStats"]

    F --> I["views series"]
    G --> J["visitors series"]

    J --> K
```

## 图 5：Events 查询

```mermaid
flowchart TD
    A["Events page"] --> B["GET /api/websites/:id/events"]
    B --> C["parseRequest schema"]
    C --> D["getQueryFilters"]
    D --> E["parseFilters"]
    E --> F["dateQuery + filterQuery + cohortQuery"]
    F --> G["getWebsiteEvents"]
    G --> H{"backend"}
    H -- Prisma --> I["join website_event + session"]
    H -- ClickHouse --> J["select from website_event"]
    I --> K["pagedRawQuery"]
    J --> K
    K --> L["rows with hasData"]
```

## 图 6：Umami 到 analytics-core 对照数据流

```mermaid
flowchart LR
    U1["Umami tracker"] --> A1["SimpleTrack tracker"]
    U2["/api/send route"] --> A2["collect HTTP adapter"]
    U3["route normalization"] --> A3["collect.Handler + Normalize"]
    U4["saveEvent direct write"] --> A4["EventBus publish"]
    A4 --> A5["ingestion.Processor"]
    U5["website_event/event_data"] --> A6["EventWriter + ClickHouse tables"]
    U6["getRealtimeData/getWebsiteEvents"] --> A7["EventQueryBuilder + EventReader"]
```
