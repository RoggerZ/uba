# 问：字段白名单是什么意思，用途是什么？

## 答

字段白名单就是“允许被查询、过滤、排序的字段清单”。只有在清单里的字段，才允许从前端参数进入后端查询。

在 Umami 里，字段白名单主要由 `references/umami/src/lib/constants.ts` 的 `FILTER_COLUMNS` 表达，例如：

- 页面路径 `path` 对应数据库列 `url_path`。
- 事件名 `event` 对应数据库列 `event_name`。
- UTM 来源 `utmSource` 对应数据库列 `utm_source`。

## 为什么需要它

没有字段白名单，前端如果传入任意字段名，后端可能会：

- 查到不该查的列。
- 生成错误 SQL。
- 被恶意参数诱导成 SQL 注入风险。
- 让高成本字段或无索引字段进入大查询，拖慢数据库。

## 举例

允许：

```text
?event=checkout_completed
```

因为 `event` 在白名单中，可以映射到 `event_name`。

拒绝或忽略：

```text
?dropTable=website_event
```

因为 `dropTable` 不在白名单中，不应该参与查询。

## 给 SimpleTrack 的启发

SimpleTrack 的 Filters UI 不应该让用户随便输入数据库字段名。产品上应提供有限字段、有限操作符，比如“事件名等于”“路径包含”“UTM 来源等于”。

## 给 analytics-core 的启发

`EventQueryBuilder` 必须内置字段白名单、排序白名单、operator enum 和分页上限。字段白名单最好用 Go 类型或常量表达，不让页面层字符串直接穿透到 SQL。
