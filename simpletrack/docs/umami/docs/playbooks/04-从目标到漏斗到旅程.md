# 从目标到漏斗到旅程

## 目标

围绕一个核心业务目标，逐步回答“有没有达成”“卡在哪一步”“用户下一步去了哪里”。

## 适用场景

- 注册、试用、付费、下载等核心转化路径
- 想定位用户在流程里的流失点
- 想把单个目标拆成更完整的行为路径

## 推荐顺序

1. 先用 Goal 定义“什么叫成功动作”
2. 再用 Funnel 定义成功动作之前的关键顺序步骤
3. 如果想看用户真实路径，再进 Journey
4. 用 Filters 切不同来源、设备或页面版本的人群比较差异
5. 最后再用 Compare 看优化前后是否真的变好

## 每步判断点

1. Goal 先回答“总体有没有发生”
2. Funnel 再回答“具体哪一步掉了”
3. Journey 再回答“用户在真实世界里到底怎么走”
4. 如果 Step 定义都还没稳定，不要急着讲 Journey 洞察
5. 如果 Goal 的分母不同，不要把两个转化率直接放在一起比较

## 常见坑

- 直接从 Journey 开始，结果因为路径过多，团队不知道该先改哪里
- Funnel 的 step 定义不一致，导致后续步骤直接掉到 0
- 把 Goal、Funnel、Journey 当成一回事，导致报表设计混乱
- 只看最终 Goal 变低，却没有回到 Funnel 检查是哪一步漏掉

## SimpleTrack 可直接借鉴的决策

- 先让用户能快速配置 Goal，再引导升级到 Funnel
- 在 Funnel 配置页里明确强调“顺序要求”和 step value 的含义
- Journey 更适合进阶洞察，不适合当新手的第一张分析页
- 转化分析链路里要固定显示当前日期范围、筛选条件和 Goal 分母，让结论可复述
