# 从过滤到细分用户

## 目标

把“看全量数据”推进到“只看我关心的那群人”，并且让分析可以重复复用。

## 适用场景

- 数据开始多起来了，想切出重点人群
- 同一套筛选条件会反复使用
- 想观察某类用户在一段时间里的表现

## 涉及能力

- [14-Filters](../14-Filters.md)
- [15-Segments](../15-Segments.md)
- [16-Cohorts](../16-Cohorts.md)
- [12-Retention](../12-Retention.md)

## 推荐顺序

1. 先用 Filter 做一次性分析
2. 确认筛选条件值得复用后，再存成 Segment
3. 如果要看“某批用户后续表现”，再升到 Cohort
4. 当目标变成“这些人之后还回来没有”，转到 Retention

## 每步判断点

1. Filter 适合临时切片
2. Segment 适合稳定人群定义
3. Cohort 适合带时间维度的人群观察
4. Retention 适合回答“回来没回来”，不是回答“今天谁贡献最高”

## 常见坑

- 把所有筛选都提前保存成 Segment，最后列表很快失控
- 用 Retention 去回答转化问题，结果读者看不懂
- Cohort 定义太宽泛，导致结果没有解释力

## SimpleTrack 可直接借鉴的决策

- 先把 Filter 做强，再决定是否需要 Segment 和 Cohort
- Segment 要有清晰命名和说明，不然团队成员不知道它代表什么
- Cohort 应该作为“进阶分析”出现，避免在 MVP 里吓到新用户
