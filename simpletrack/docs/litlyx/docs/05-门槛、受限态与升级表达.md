# Litlyx 门槛、受限态与升级表达

> 目标：把 Litlyx 里“用户现在不能继续做”的状态拆清楚，区分哪些是清晰门槛、哪些是可借鉴的升级表达、哪些会造成误解。

## 1. 当前看到的受限态类型

| 类型 | Litlyx 表现 | 当前证据 | 结论 |
| --- | --- | --- | --- |
| 明确付费门槛 | SEO premium gate + Upgrade CTA | `P04-S06` `REF-07` | 表达清楚，用户知道这是未解锁能力 |
| 模板先行、生成受限 | Reports 模板页、Sample 可预览、正式生成 disabled | `P04-S04` `P04-S05` `P04-S07` | 高级价值先露出，正式生成被套餐限制 |
| FAQ 解释边界 | Plans FAQ 解释 free trial 相关问题 | `P05-S09` | 用 FAQ 补足套餐和试用期心智 |
| 创建动作未执行 | Shareable links 空态、Create link 可见 | `P05-S10` | 入口存在，但创建会改变云端访问状态，本轮未触发 |
| 原因不清的 loading | Members 直接访问后停在 loading / contact owner | `P05-S11` | 反馈弱，不如 SEO 和 Reports 清楚 |
| 外链式“继续配置” | Product 的 `Setup events` 打开外部文档 | `P03-S07` | 功能可用，但 CTA 去向不够直观 |

## 2. Reports：样张先行，生成受限

Reports 的表达分成三层：

1. 模板中心让用户知道有哪些报告类型
2. `Sample` 让用户先看成品样张
3. 正式报告卡片和底部 `Generate report` 在当前 Free trial 下 disabled

这是一种值得借鉴的高级能力表达：先解释价值，再说明门槛。

需要注意的是，Litlyx 当前没有把“为什么 disabled”解释得特别完整。SimpleTrack 如果借这个模式，最好直接说明：

- 当前套餐不支持
- 需要选择报告类型
- 需要升级或联系 owner

## 3. SEO：门槛表达最清楚

SEO 页的 premium gate 是当前最明确的受限态：

- 用户知道能力存在
- 用户知道当前不能用
- 用户知道下一步是 `Upgrade`

这比空白页、loading 或灰态按钮更好。SimpleTrack 做高级能力时，应优先采用这种清晰门槛表达。

## 4. Plans：用 FAQ 补足试用期边界

Plans 页不是单纯价格表，它把 Personal / Business、FAQ 和试用期说明放在一起。

这个设计的价值在于：

- 减少用户猜测
- 把功能限制和升级预期讲清楚
- 让套餐页同时承担转化和解释作用

## 5. Shareable links：入口清楚，但动作有风险

Shareable links 当前状态是 `0 active`，可以看到 public / protected link 等配置入口。

这说明 Litlyx 把外部只读分享单独建模了。但本轮没有点击 `Create link`，因为这会创建新的访问入口，属于会改变云端访问状态的动作。

对 SimpleTrack 来说，值得借鉴的是“外部只读分享”和“内部成员协作”分开；不应直接照搬的是在调研和自动化里无提示创建真实分享链接。

## 6. Members：受限态表达最弱

Members 是当前最不清楚的受限态：

- 侧栏里灰态
- 直接访问 `/members` 后长时间 loading
- 文案只提示 contact project owner
- 没有明确说明是权限限制、套餐限制还是接口异常

这个状态不适合照搬。SimpleTrack 如果有成员协作门槛，应该直接说明原因：

- 当前套餐不支持
- 当前用户不是 owner
- 当前 workspace 未启用成员功能
- 系统异常，请稍后重试

## 7. `Setup events`：不是不能用，而是去向容易误判

`Setup events` 本身不是 disabled，也不是功能缺失。当前验证表明它会打开 Custom Events 文档新标签页。

问题在于：这个 CTA 文案像站内配置入口，但真实动作是外部文档跳转。用户如果没注意到新标签页，容易以为按钮无响应。

SimpleTrack 里类似入口应写得更明确，例如：

- `Read custom events docs`
- `Open setup guide`
- `Learn how to send custom events`

## 8. 对 SimpleTrack 的建议

### 值得借鉴

- 高级能力先给样张或示例，再解释门槛
- premium gate 要带明确下一步
- 套餐页和 FAQ 应直接解释能力边界
- 外部只读分享和内部成员协作分开

### 不要照搬

- 只有 loading 的成员受限态
- 灰态按钮没有解释原因
- 看起来像站内配置、实际打开外链的 CTA
- 会改变云端访问状态的创建动作缺少明确确认
