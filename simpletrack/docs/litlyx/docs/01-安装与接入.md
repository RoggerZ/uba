# Litlyx 安装与接入

## 当前结论

Litlyx 把首次使用的重点放在“尽快接入”，而不是“先浏览空 dashboard”。

登录后默认落点就是 `Web` 安装页，同屏提供：

- `Script`
- `Tag (GTM)`
- 框架快捷入口
- `Workspace ID`
- AI setup prompt
- `Verify Installation`

## 为什么这条链路有效

它把新手最需要的动作压缩成了一条很短的路：

1. 复制代码
2. 根据框架或 AI prompt 完成接入
3. 回后台验证

对早期产品来说，这比把用户直接送进空分析页更稳。

## 回查机制

`Settings / General` 再次暴露：

- Workspace name
- Workspace ID
- Script
- Delete workspace

这说明 Litlyx 认为“安装信息回查”是高频需求，而不是一次性动作。

## 对 SimpleTrack 的启发

- 默认首页优先做安装入口
- 安装信息至少要在两个位置可回查
- `Verify Installation` 这类验收入口要显式存在
- AI 可以服务接入，但不应替代清晰的脚本和 GTM 路径
