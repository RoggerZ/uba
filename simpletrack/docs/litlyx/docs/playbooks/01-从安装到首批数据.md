# 从安装到首批数据

## 目标

把一个新的 SimpleTrack 工作区，从“用户刚登录”推进到“确认真实事件已经进入 Product 和 Raw Events”。

## 适用场景

- 第一次接入 tracker
- 需要做首批数据验收
- 需要设计更顺的 onboarding

## 推荐顺序

1. 登录后默认进入安装入口，而不是空 dashboard
2. 在安装页提供 Script / GTM / 框架入口 / AI prompt
3. 在 Settings 再次暴露 Workspace ID 和安装代码，便于回查
4. 先让用户完成 pageview 和一个最小事件
5. 先看 Raw Events 确认入库，再看 Product 聚合结果
6. 如果真实数据暂时不足，再用 `Show test data` 做教育，而不是把它当真实验证

## 每步判断点

1. 如果用户刚登录还没接入，就不要先把注意力带去复杂报表
2. 如果事件链路没确认，不要急着讨论漏斗、报告和 AI 洞察
3. 如果聚合图暂时没更新，先看 Raw Events，不要直接判定接入失败
4. `Show test data` 只解决理解问题，不解决验收问题

## 常见坑

- 首屏先给空分析页，用户不知道下一步做什么
- 安装代码只出现在一个页面，回查路径太长
- 只看 Product 聚合，不看明细表，结果排障效率很低
- 把示例数据误当成真实链路验证

## SimpleTrack 可直接借鉴的决策

- 默认首页优先做安装入口
- 把 `Workspace ID + 安装代码 + 验证入口` 放进一条连续链路
- 同时提供聚合页和事件明细页
- 用示例数据帮用户理解“未来会看到什么”，但验收仍以真实事件为准
