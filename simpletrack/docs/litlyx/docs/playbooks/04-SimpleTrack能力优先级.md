# SimpleTrack 能力优先级

## 目标

把 Litlyx 当前最值得借的部分拆成分阶段路线，避免一开始把产品做成“功能很多，但第一天没人用得起来”。

## 推荐顺序

1. 先做接入成功和首批数据可见
2. 再做分析教育和增长闭环
3. 然后做治理、分享和协作边界
4. 最后补高级报告生成和更完整权限体系

## MVP

- 登录后默认安装入口
- Script / GTM / 框架入口 / AI 安装 prompt
- Settings 里的安装信息回查
- Product 骨架页
- `Show test data`
- Raw Events 明细表
- 最小事件验证链路

## 增强版

- Marketing 页
- `Generate UTM link`
- Reports 模板中心
- Reports `Sample` 样张预览
- AI 任务型 prompt 模板
- Shields 的基础治理能力

## 后置能力

- Reports 正式 PDF 生成链路
- Shareable links 创建与撤销
- Members 邀请与更完整权限模型
- Domain data 删除与 sanitization 审计
- 更细的套餐、门槛和商业化控制

## 每步判断点

1. 没有首批数据可见性，就不要先做复杂增长闭环
2. 没有稳定事件语义，就不要先堆报告生成类型
3. 没有明确分享场景，就不要过早做复杂成员和权限体系
4. 治理能力一旦上线，就必须同时考虑风险提示和审计

## Litlyx 给我们的提醒

- 用户第一天更需要“成功接入”，不是“看到所有功能”
- 示例数据和样张能显著降低理解成本
- 受限态一定要解释清楚，不然就像产品坏了
- 治理、分享、协作要分层，不要混在同一页
