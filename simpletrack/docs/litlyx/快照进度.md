## 重点产品发现

1. `Web` 首页是安装中心，不是 dashboard。
2. `Show test data` 是很强的教育机制，`Product` 和 `Marketing` 都适用。
3. `tracking-demo` 已针对当前工作区完成一轮真实验证：浏览器脚本成功加载，后台 `Product` 与 `Raw Events` 均已出数。
4. `Setup events` 已完成复核：前端 chunk 和 DOM 都显示它是 `https://docs.litlyx.com/custom-events` 外链，点击会打开 `Custom Events - Litlyx Docs` 新标签页。
5. `Reports` 的 `Sample` 预览不是点卡片本体，而是点卡片右上角单独的 `Sample` 按钮；当前 Free trial 下正式报告卡片和 `Generate report` 都处于 disabled。
6. `Settings / Domains` 不是弱空态，而是域名数据删除和 Sanitization 工具入口，本轮没有点击任何会删除数据的按钮。
7. `Shareable links` 是独立的只读分享页，当前 `0 active`；`Create link` 会创建访问入口，本轮没有触发。
8. `Members` 直接访问后停在 loading / contact owner 态，反馈弱于 SEO 和 Reports 这类明确门槛页。
9. `Litlyx功能矩阵.md` 已把功能模块、关键工作流、权限风险和 SimpleTrack 借鉴优先级整理成查表式材料。
10. `docs/` 层已补上 Litlyx/Umami 取舍对照与 4 份 playbook，更适合直接进入 SimpleTrack 范围讨论。
11. `docs/落地评审清单.md` 已把接入、分析、增长、治理、分享、商业化拆成可逐项核对的评审表。
12. `docs/功能打通矩阵.md` 与 `docs/执行与复验手册.md` 已补齐后续接力所需的状态表和操作手册。
13. `docs/数据模型与事件字典.md` 与 `docs/真实业务数据方案.md` 已把当前真实样本和下一阶段补样边界拆开记录。
14. `docs/00` 到 `docs/05` 已把 Litlyx 拆成能力地图、安装接入、Product、增长闭环、治理协作、门槛升级六篇模块专题。
15. `docs/SimpleTrack实施路线图.md` 已把 Litlyx 结论整理成阶段、交付物、验收标准和升级条件。
16. `docs/SimpleTrack产品需求草案.md` 已把 Litlyx 启发转成 PRD 起点，覆盖目标、范围、用户故事、功能需求和验收标准。
