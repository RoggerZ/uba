# SimpleTrack 原型驱动的页面数据契约与接口草案

> 目的：基于 `simpletrack/prototype/simpletrack-umami-inspired/` 下的原型页面，冻结第一版后端页面 contract，避免在开发阶段继续一边写前端一边改产品结构。

---

## 一、适用范围

本草案覆盖以下原型页面：

- `login.html`
- `create-site.html`
- `install.html`
- `first-data.html`
- `index.html`
- `events.html`
- `funnels.html`
- `insights.html`
- `team.html`
- `settings.html`

原则：

- 页面只拿自己需要的数据
- 优先页面级/BFF 级接口，不让前端直接拼接过多分析参数
- 先冻结 MVP 页面 contract，再决定内部如何复用 `xwl_bi`

## 三、页面 contract

## 1. `login.html`

页面目的：

- 用户进入产品
- 启动 onboarding 流程

前端需要：

- 无复杂数据依赖
- 只需要提交登录表单

建议接口：

### `POST /api/auth/login`

请求：

```json
{
  "email": "founder@acme.io",
  "password": "string"
}
```

响应：

```json
{
  "token": "jwt_or_session_token",
  "user": {
    "id": "usr_001",
    "email": "founder@acme.io",
    "name": "Founder"
  },
  "next_step": "create_site"
}
```

---

## 2. `create-site.html`

页面目的：

- 创建第一个 site
- 记录安装上下文

建议接口：

### `POST /api/sites`

请求：

```json
{
  "workspace_name": "Acme Workspace",
  "site_name": "Acme marketing site",
  "domain": "acme.io",
  "framework": "nextjs"
}
```

响应：

```json
{
  "workspace": {
    "id": "ws_001",
    "name": "Acme Workspace",
    "plan_name": "Pro trial",
    "weekly_insight_enabled": true
  },
  "site": {
    "id": "site_001",
    "name": "Acme marketing site",
    "domain": "acme.io",
    "framework": "nextjs",
    "snippet_token": "site_01HZX"
  }
}
```

---

## 3. `install.html`

页面目的：

- 告诉用户如何安装
- 让用户完成“复制 snippet”

建议接口：

### `GET /api/sites/:site_id/install`

响应：

```json
{
  "site": {
    "id": "site_001",
    "name": "Acme marketing site",
    "domain": "acme.io",
    "framework": "nextjs"
  },
  "snippet": {
    "html": "<script async src=\"https://cdn.simpletrack.io/st.js\" data-site=\"site_01HZX\" data-framework=\"nextjs\"></script>"
  },
  "install_hint": {
    "title": "Install in your root layout",
    "body": "Paste the snippet into the document head or framework root layout."
  }
}
```

### `POST /api/sites/:site_id/install/copied`

用途：

- 记录用户复制 snippet 行为

请求：

```json
{
  "copied_at": "2026-04-03T10:00:00Z"
}
```

响应：

```json
{
  "ok": true
}
```

## 四、内部实现建议

这份 contract 不要求你把 `xwl_bi` 的原始接口直接暴露给前端。

建议做一层 BFF：

- `core-api`
  承担：
  - 账号
  - site/workspace/team
  - onboarding
  - insight orchestration
  - 页面聚合接口

`xwl_bi` 相关能力放在内部：

- ingest
- analysis
- metadata
- saved reports

也就是：

- 前端只认页面接口
- BFF 再去调用底层分析引擎

---

## 五、冻结建议

在正式进入后端开发前，至少先冻结这 3 件事：

1. 页面集合是否就是当前这 10 个页面
2. `Insights` 是否单独成页
3. `Team` 是否保留独立页

如果这 3 件事不冻结，后端 contract 很容易持续变动。
