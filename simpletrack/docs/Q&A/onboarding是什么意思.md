# onboarding 是什么意思

## Q：onboarding 是什么意思？

A：onboarding 指新用户第一次进入产品后的引导流程。它不是单纯的欢迎页，而是帮助用户完成“第一次成功使用”的路径。

## Q：在 SimpleTrack 里 onboarding 应该包含什么？

A：SimpleTrack 的 onboarding 应该围绕“让数据管道活起来”设计：

1. 创建 workspace 或默认工作区。
2. 创建第一个 site。
3. 生成安装 snippet。
4. 指引用户把 snippet 放到网站。
5. 检查是否收到 pageview。
6. 引导触发一个自定义事件。
7. 跳转到 Realtime 或 Events 查看数据。

## Q：onboarding 和普通设置页有什么区别？

A：设置页是长期管理入口，onboarding 是首次达成关键成功动作的流程。SimpleTrack 的首次成功动作不是“注册完成”，而是“第一个事件进入系统并被用户看见”。

## Q：P1 onboarding 要做到多复杂？

A：不复杂。P1 只需要一个清晰的 checklist 或 step flow，覆盖创建站点、安装 snippet、接收 pageview、查看事件。不要在 P1 加团队邀请、套餐升级、AI 周报配置等额外步骤。
