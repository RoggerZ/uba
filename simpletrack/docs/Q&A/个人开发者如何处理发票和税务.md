# 个人开发者如何处理发票和税务

## Q：我是个人开发者，发票和税务处理是什么意思？

A：发票处理指客户付款后，谁给客户开具收据或发票，以及发票上显示谁是卖方。税务处理指不同国家或地区的销售税、VAT/GST、所得申报等问题由谁计算、代收、申报或承担。

## Q：如果使用 MoR 平台，会发生什么？

A：在 MoR 模式下，平台通常作为面向客户的卖方，负责向客户收款、生成发票或收据、处理销售税/VAT/GST、退款和拒付。你作为开发者再从平台获得结算收入。

这能显著降低你直接面对全球客户时的税务和发票复杂度，但不代表你个人所在地的收入申报义务消失。

## Q：如果使用 Stripe，会发生什么？

A：Stripe 通常是支付处理平台，不是交易卖方。你或你的主体更像直接卖方，可能需要自己处理客户发票、税务登记、销售税/VAT/GST、退款政策和收入申报。Stripe 有 Tax、Billing、Invoicing 等工具，但责任边界仍不同于 MoR。

## Q：个人开发者早期更建议哪条路？

A：早期更建议优先评估 Lemon Squeezy、Paddle、Polar 这类 MoR 或类 MoR 路线，因为它们能把发票、销售税和交易责任集中到平台侧处理。等 SimpleTrack 收入稳定，再评估是否成立主体、接 Stripe 或做更复杂的企业支付。

## Q：这是不是法律或税务建议？

A：不是。这只是产品和工程选型层面的解释。真正上线收费前，需要按你的所在地、收款平台、主体类型和收入规模咨询专业税务或合规意见。
