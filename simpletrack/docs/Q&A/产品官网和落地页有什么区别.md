# 产品官网和落地页有什么区别

## Q：P1 需要像 Umami 官网那样的页面，这叫落地页吗？

A：如果只有一个宣传转化页面，可以叫落地页（Landing Page）。但 SimpleTrack P1 需要的不只是单页宣传，而是一个 **产品官网 / Marketing Site / 公开站点**。

它至少包含：

- 产品介绍：SimpleTrack 是什么、解决什么问题、适合谁。
- 定价/订阅入口：套餐、计费周期、升级入口。
- docs/quickstart：如何创建站点、复制 snippet、验证 pageview 和自定义事件。
- 登录/注册入口：进入 SaaS 控制台。

所以文档里优先使用“产品官网 / Marketing Site / 公开站点”。“落地页”可以作为其中的首页或转化页，但不是完整范围。

## Q：为什么 P1 就要做公开站点？

A：因为 SimpleTrack 是面向外部客户的 SaaS，不是内部工具。P1 如果只有控制台，没有公开产品入口，用户就不知道产品价值、如何购买、如何接入。

公开站点和数据管道一起完成，能让 P1 形成最小商业闭环：

1. 用户看懂产品。
2. 用户看到定价或订阅入口。
3. 用户注册进入控制台。
4. 用户按 quickstart 安装 tracker。
5. 用户在 Realtime 和 Events 看到数据。

## Q：这部分应该放在哪里实现？

A：优先放在 SaaS 模板自带的 marketing/docs app 里。以 Supastarter 为例，本地源码已经有：

- `apps/marketing`
- `apps/docs`
- `apps/saas`

这正好对应公开官网、文档和登录后的产品控制台。
