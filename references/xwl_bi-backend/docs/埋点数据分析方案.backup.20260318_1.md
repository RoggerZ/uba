# 短剧APP埋点数据分析方案

本文档基于 `shortAAA` (用户中心), `shortPortal` (C端接口), `short-cms` (内容管理) 项目代码分析，制定针对短剧业务的埋点数据分析方案。

## 1. 梳理用户属性、事件、事件属性

### 1.1 用户属性 (User Properties)

用户属性用于描述用户的特征，在所有事件中通用。

| 属性名称 | 标识符 | 类型 | 描述/来源 |
| :--- | :--- | :--- | :--- |
| 用户ID | `user_id` | String | 登录用户的唯一标识 (`shortAAA` claims.Uid) |
| 登录名 | `login_name` | String | 用户登录账号 |
| 用户类型 | `cust_type` | Integer | 0: 游客, 1: 注册用户 |
| VIP等级 | `vip_level` | Integer | 用户VIP等级 (0: 普通, 1: VIP/分享达人) |
| 账户余额 | `gold_coin` | Integer | 当前金币/积分余额 |
| 登录方式 | `provider` | String | Google, Apple, Facebook, Tiktok, Email 等 |
| 账号绑定 | `is_bound` | Boolean | 是否已绑定第三方账号 |
| 设备ID | `device_id` | String | 对应Header `X-Dev` |
| 设备品牌 | `device_brand` | String | 对应Header `X-Brand` |
| 设备型号 | `device_model` | String | 对应Header `X-model` |
| 系统版本 | `os_version` | String | 对应Header `X-OsVersion` |
| 安装渠道 | `referrer` | String | 对应Header `X-Referrer` (归因来源) |
| 注册时间 | `signup_time` | Timestamp | 用户首次注册时间 |
| 语言偏好 | `language` | String | 用户首选语言 (`X-Lang`) |

### 1.2 核心事件与属性 (Events & Properties)

#### (1) 应用启动 (App Launch)
*   **触发时机**: 用户冷启动或后台唤醒 App。
*   **属性**:
    *   `launch_type`: 启动类型 (cold/resume)
    *   `network_type`: 网络类型 (WiFi/4G/5G)

#### (2) 用户登录/注册 (Login/Register)
*   **触发时机**: 用户完成登录或注册流程。
*   **属性**:
    *   `method`: 登录方式 (google, apple, facebook, guest)
    *   `is_success`: 是否成功
    *   `fail_reason`: 失败原因

#### (3) 浏览剧集详情 (View Drama Detail)
*   **触发时机**: 用户进入剧集详情页 (`shortPortal` - `GetByEpisodeId`).
*   **属性**:
    *   `drama_id`: 剧集ID (`program_id`)
    *   `drama_title`: 剧集名称
    *   `drama_type`: 剧集类型 (short/long)
    *   `tags`: 题材标签 (Genre)
    *   `is_exclusive`: 是否独家
    *   `total_episodes`: 总集数

#### (4) 观看视频 (Watch Video)
*   **触发时机**: 用户开始播放某一集视频 (`shortPortal` - `HistoryHandler`).
*   **属性**:
    *   `drama_id`: 所属剧集ID
    *   `video_id`: 视频/分集ID
    *   `episode_index`: 第几集 (`sort`)
    *   `duration`: 观看时长 (秒)
    *   `progress`: 播放进度
    *   `is_paid_content`: 该集是否为付费集
    *   `is_unlocked`: 用户是否已解锁该集

#### (5) 付费/充值 (Purchase)
*   **触发时机**: 用户充值金币或购买会员。
*   **属性**:
    *   `product_id`: 商品ID
    *   `amount`: 金额
    *   `currency`: 货币类型
    *   `payment_method`: 支付方式 (GooglePay, ApplePay)
    *   `pay_status`: 支付状态

#### (6) 解锁剧集 (Unlock Episode)
*   **触发时机**: 用户消耗金币解锁付费集。
*   **属性**:
    *   `drama_id`: 剧集ID
    *   `video_id`: 视频ID
    *   `cost_coins`: 消耗金币数
    *   `balance_after`: 剩余金币数

#### (7) 互动行为 (Interaction)
*   **触发时机**: 点赞、收藏、分享。
*   **属性**:
    *   `action_type`: like (点赞), favorite (收藏), share (分享)
    *   `target_type`: drama (剧集), video (视频)
    *   `target_id`: 目标ID
    *   `platform`: 分享到的平台 (Tiktok, Whatsapp等)

#### (8) 搜索 (Search)
*   **触发时机**: 用户执行搜索。
*   **属性**:
    *   `keyword`: 搜索关键词
    *   `result_count`: 结果数量

## 2. 行为分析：事件分析 (Event Analysis)

针对短剧APP，核心关注**内容消费**与**商业化转化**。

### 2.1 推荐分析事件
1.  **用户增长分析 (User Growth)**:
    *   **分析事件**: `Login` (登录), `Register` (注册)
    *   **分析指标**: 注册转化率 (Register/Login), 每日新增注册用户数。
    *   **维度**: `referrer` (渠道), `os_version` (系统)。
    *   **目的**: 监控新用户获取效率和注册流程的顺畅度。

2.  **视频完播率分析**:
    *   **分析事件**: `Watch Video` (观看视频)
    *   **分析指标**: 平均观看时长、完播次数占比。
    *   **维度**: `episode_index` (第几集), `drama_id` (剧名), `referrer` (渠道)。
    *   **目的**: 判断哪一集流失率最高，优化剧情节奏。

3.  **付费转化分析**:
    *   **分析事件**: `Unlock Episode` (解锁剧集)
    *   **分析指标**: 解锁次数、消耗金币总数。
    *   **维度**: `drama_id` (剧集), `is_exclusive` (是否独家)。
    *   **目的**: 识别最吸金的剧集。

4.  **充值活跃度**:
    *   **分析事件**: `Purchase` (充值)
    *   **分析指标**: 充值总金额 (GMV), 充值人数 (PayUser)。
    *   **维度**: `provider` (登录方式), `referrer` (投放渠道)。

### 2.2 用户分群配置 (User Segmentation)
*   **高价值用户 (High Value)**: 过去30天 `Purchase` 事件的总金额 > $50。
*   **沉浸式用户 (Engaged)**: 过去7天 `Watch Video` 时长 > 60分钟。
*   **潜在流失用户 (Risk)**: 过去7天无 `Launch App` 事件，且曾有充值记录。
*   **剧集偏好群**: 过去30天 `View Drama Detail` 中 `tags` 包含 "逆袭" 或 "豪门" 的用户。

## 3. 行为分析：留存分析 (Retention Analysis)

关注用户是否持续回到APP消费内容。

### 3.1 推荐分析场景
1.  **新用户次留/七留**:
    *   **初始事件**: `Register` (注册) 或 `Launch App` (首次启动)。
    *   **回访事件**: `Launch App` (启动) 或 `Watch Video` (看剧)。
    *   **目的**: 评估投放渠道质量 (ROI)。

2.  **游客转注册留存**:
    *   **初始事件**: `Login` (UserType=0 游客登录)。
    *   **回访事件**: `Register` (后续产生注册行为)。
    *   **目的**: 分析游客体验产品后转化为注册用户的比例。

3.  **付费用户留存**:
    *   **初始事件**: `Purchase` (首次充值)。
    *   **回访事件**: `Unlock Episode` (复购/解锁) 或 `Watch Video`。
    *   **目的**: 评估付费用户的生命周期价值 (LTV)。

### 3.2 用户分群配置
*   **渠道新客**: `referrer` 为 "Tiktok Ads" 且 `signup_time` 在过去7天内的用户。
*   **首充用户**: 仅发生过1次 `Purchase` 事件的用户。

## 4. 行为分析：漏斗分析 (Funnel Analysis)

分析用户从接触到核心转化的路径转化率。

### 4.1 推荐分析场景
1.  **注册转化漏斗 (Registration Conversion)**:
    *   **步骤 1**: `AppLaunch` (应用启动)
    *   **步骤 2**: `Login` (发起/完成游客登录)
    *   **步骤 3**: `Register` (完成注册)
    *   **目的**: 分析从启动到最终注册的流失情况，识别“只逛不买（不注册）”的用户群体。

2.  **核心付费漏斗 (Paywall Conversion)**:
    *   **步骤 1**: `View Drama Detail` (浏览剧集)
    *   **步骤 2**: `Watch Video` (开始观看免费集)
    *   **步骤 3**: `Unlock Episode` (点击解锁/遇到付费点) -> *触发充值引导*
    *   **步骤 4**: `Purchase` (完成充值)
    *   **步骤 5**: `Unlock Episode` (成功解锁)
    *   **目的**: 找到转化率断崖点（如：是剧不好看没看到付费点，还是付费引导页太贵）。

3.  **注册转化漏斗**:
    *   **步骤 1**: `Launch App` (启动)
    *   **步骤 2**: `View Profile/Mine` (点击我的)
    *   **步骤 3**: `Login` (发起登录)
    *   **步骤 4**: `Login Success` (登录成功)

### 4.2 用户分群配置
*   **卡点用户**: 完成了 步骤2 (观看) 但未进行 步骤3 (解锁) 的用户 -> 推送限时优惠券。
*   **支付失败用户**: 触发了 步骤4 (发起支付) 但 `pay_status` != success 的用户。

## 5. 行为分析：路径分析 (Path Analysis)

探索用户的自然行为流向，发现非预期的行为路径。

### 5.1 推荐分析场景
1.  **内容发现路径**:
    *   **起始事件**: `Launch App`。
    *   **观察路径**: 用户启动后是先去 `Search` (搜索)，还是浏览 `Home Feed` (推荐流)，还是去 `History` (历史记录)。
    *   **目的**: 优化首页布局。如果大量用户直接搜索，说明推荐不准；如果直接去历史，说明追剧粘性高。

2.  **付费后行为路径**:
    *   **起始事件**: `Purchase` (充值成功)。
    *   **观察路径**: 充值后是立即 `Unlock Episode` (解锁当前剧)，还是去寻找其他剧集，或者直接退出。

### 5.2 用户分群配置
*   **搜索型用户**: 起始路径主要为 `Search` 的用户。
*   **推荐流用户**: 主要在 `Home Feed` 产生交互的用户。
