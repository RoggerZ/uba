# 短剧项目用户分析规划

## 1. 概述
本文档旨在定义短剧项目的用户画像体系、分群策略及生命周期管理，支持游客模式、金币与订阅混合变现模式下的精细化运营。

## 2. 用户属性体系 (User Properties)

### 2.1 基础属性
| 属性名 | 属性标识 (Property Key) | 描述 | 更新频率 |
| :--- | :--- | :--- | :--- |
| 用户ID | `user_id` | 全局唯一标识 (游客时为设备ID生成伪ID, 登录后合并) | 注册/启动时 |
| 是否游客 | `is_guest` | true/false | 每次启动/登录更新 |
| 注册时间 | `signup_time` | 用户首次注册时间 (游客为空) | 一次性 |
| 地域 | `city`, `province`, `country` | 用户所在地理位置 | 每次启动更新 |
| 设备型号 | `device_model` | 如 iPhone 13, Huawei P40 | 每次启动更新 |
| 渠道来源 | `channel` | 用户获客渠道 | 注册/首次启动时记录 |

### 2.2 业务属性
| 属性名 | 属性标识 (Property Key) | 描述 | 更新频率 |
| :--- | :--- | :--- | :--- |
| 会员状态 | `vip_status` | none, active, expired | 实时 |
| 会员到期日 | `vip_expire_date` | 会员到期时间 | 订阅变动时更新 |
| 剩余金币 | `coin_balance` | 用户当前账户金币余额 | 实时 |
| 累计充值金额 | `total_pay_amount` | 历史总充值金额 (LTV参考) | 每次支付后累加 |
| 观看剧集数 | `watched_drama_count` | 历史观看过的不同剧集数量 | 每日更新 |
| 最近观看剧集 | `last_watch_drama` | 最近一次观看的剧集ID | 每次播放更新 |
| 首充时间 | `first_pay_time` | 第一次充值的时间 | 一次性 |
| 最近充值时间 | `last_pay_time` | 最近一次充值的时间 | 每次支付更新 |

## 3. 用户分群策略 (User Segmentation)

### 3.1 基于身份与状态
*   **纯游客**：`is_guest=true` 且 `total_pay_amount=0`。
*   **未注册付费游客**：`is_guest=true` 且 `total_pay_amount > 0` (风险群体，需引导绑定)。
*   **注册未付费**：`is_guest=false` 且 `total_pay_amount=0`。
*   **金币付费用户**：有金币充值记录。
*   **订阅会员用户**：`vip_status=active`。

### 3.2 基于价值 (RFM模型)
*   **高价值用户 (大R)**：累计充值 > 500元，或长期订阅会员。
*   **活跃白嫖党**：周活跃天数 > 3，日均观看时长 > 30min，但无付费。
*   **流失付费用户**：曾有付费记录，但最近 14 天无活跃。

### 3.3 基于内容偏好
*   **偏好类型**：根据 `play_duration` 加权计算出的偏好类型 (如 霸总, 逆袭, 悬疑)。
*   **完播意愿**：平均剧集完播率 > 50% 的用户 (高质量内容消费者)。

## 4. 用户生命周期管理 (User Lifecycle Management)

| 阶段 | 定义 | 核心目标 | 运营策略 |
| :--- | :--- | :--- | :--- |
| **游客期** | 未登录，浏览 For You 或首页 | 引导沉浸观看，转化为注册/付费 | 免费集数试看，新人限时礼包(需注册领取) |
| **转化期** | 完成注册 或 首次充值 | 提升首充/首订转化率 | 首充优惠，订阅首月折扣 |
| **成长期** | 持续观看，偶尔付费 | 培养付费习惯，提升ARPU | 剧集更新提醒，金币余额不足提醒 |
| **成熟期** | 稳定订阅 或 高频充值 | 延长LTV，防止流失 | VIP专属剧集，积分商城，生日关怀 |
| **流失期** | 活跃度显著下降 | 召回用户 | 热门新剧推送，回归礼包(金币/体验卡) |
