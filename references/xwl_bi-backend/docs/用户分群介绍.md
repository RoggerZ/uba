# 用户分群 (User Segmentation)

## 1. 什么是用户分群？ (专业概念)

用户分群（User Segmentation）是指根据用户的属性特征或行为特征，将总体用户划分为若干个具有共同特征的细分群体。

在数据分析系统中，分群通常有两种存在形式：
1.  **静态分群 (Snapshot)**: 一个固定的用户ID列表。例如“2023年双11当天购买过商品的用户”。这个名单一旦生成，就不会自动改变，除非手动更新。
2.  **动态分群 (Dynamic)**: 一组筛选规则。例如“最近30天活跃的用户”。这个群体的名单是随着时间推移自动变化的。

> **注意**: 目前咱们系统（xwl_bi）主要支持的是**静态分群**，即基于某次分析结果保存下来的用户列表快照。

## 2. 通俗易懂的解释 (大白话)

如果把所有用户比作**一大池子鱼**，那么用户分群就是**用不同的网把鱼捞出来，分养在不同的鱼缸里**。

### 举个栗子：超市会员管理 🛒

想象你是一家大型超市的经理：

*   **全体用户**: 所有进过超市的人。
*   **分群 A (土豪组)**: 单次消费满 1000 元的人。
    *   *对策*: 发送高端红酒优惠券。
*   **分群 B (宝妈组)**: 经常购买尿不湿和奶粉的人。
    *   *对策*: 推荐婴儿用品打折信息。
*   **分群 C (流失组)**: 以前经常来，但最近一个月没来过的人。
    *   *对策*: 发短信“亲，想你了，送你一桶油回来看看吧”。

**总结一句话**：
用户分群就是**把人分类**，为了**对症下药**。

---

## 3. 为什么要做用户分群？ (价值)

不做分群的运营就是“广撒网”，效率低且费钱。做了分群可以实现“精准打击”。

1.  **精细化运营**:
    *   对不同的人说不同的话。你不能给“单身男青年”推销“孕妇装”，这不仅没用，还会让用户觉得你很烦。
2.  **洞察差异**:
    *   你可以对比分析：“为什么‘土豪组’的留存率比‘流失组’高那么多？”从而发现产品的核心价值点。
3.  **验证假设**:
    *   你可以把一群人单独拎出来做实验（A/B Test），看看新功能是不是只对这群人有效。

## 4. 在系统中怎么用？

在我们的 BI 系统中，用户分群通常是这样产生的：

1.  **先分析**: 你先在“漏斗分析”里发现了一群人，比如“加了购物车但最后没付款的人”。
2.  **存分群**: 你觉得这群人很有挽回价值，于是点击“保存为分群”，起名叫“放弃支付待挽回组”。
3.  **再应用**:
    *   你可以去“消息推送”系统，专门给这群人发优惠券。
    *   或者在“事件分析”里，专门看这群人后来的行为，看他们是不是因为价格太贵才不买的。

### 常见的分群维度

*   **基础属性**: 性别、年龄、城市、手机型号（比如只看 iPhone 用户）。
*   **行为特征**: 购买次数、累计消费金额、最近一次访问时间（RFM模型）。
*   **具体行为**: 做过某件事（如点击过活动页）或没做过某件事。
