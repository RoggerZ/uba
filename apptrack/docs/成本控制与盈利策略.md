## 📉 成本优化

### 1. AI成本优化（最重要）

**当前成本**：
- GPT-4：$0.03/1K tokens
- 每次分析：~2K tokens
- 100用户/周：$24/月

**优化策略**：

**A. 使用更便宜的模型**
```go
func (s *AIService) SelectModel(dataSize int) string {
    if dataSize < 1000 {
        return "gpt-3.5-turbo" // $0.001/1K tokens
    }
    return "gpt-4-turbo" // $0.03/1K tokens
}
```
节省：70%

**B. 优化Prompt**
```go
// 优化前：2000 tokens
// 优化后：500 tokens
prompt := fmt.Sprintf(`
分析数据并给出3点建议：
数据：DAU %d, 留存 %.1f%%
要求：简洁，<200字
`, dau, retention)
```
节省：75%

**C. 批量处理**
```go
// 每周一次性处理所有用户
func (s *AIService) BatchProcess() {
    users := s.repo.GetActiveUsers()
    for _, user := range users {
        s.GenerateInsight(user.ID)
        time.Sleep(1 * time.Second)
    }
}
```
节省：20%

**综合优化后**：
- 原成本：$50/月（100用户）
- 优化后：$10/月
- **节省80%**

### 2. 数据库成本优化

**ClickHouse优化**：
```sql
-- 使用物化视图
CREATE MATERIALIZED VIEW daily_stats AS
SELECT
    toDate(timestamp) as date,
    app_id,
    count() as events,
    uniq(user_id) as users
FROM events
GROUP BY date, app_id;

-- 数据分区
ALTER TABLE events
PARTITION BY toYYYYMM(timestamp);

-- 自动删除旧数据
ALTER TABLE events
DROP PARTITION '202401';
```

**PostgreSQL优化**：
```sql
-- 索引优化
CREATE INDEX idx_events_app_time
ON events(app_id, timestamp DESC);

-- 数据压缩
ALTER TABLE events
SET (toast_compression = lz4);
```

节省：40%存储空间

## 🚨 防止成本失控

### 1. 恶意用户防护

```go
// IP限流
func (m *IPRateLimiter) Limit() gin.HandlerFunc {
    return func(c *gin.Context) {
        ip := c.ClientIP()
        key := fmt.Sprintf("ip:%s:%d", ip, time.Now().Unix()/60)

        count, _ := m.redis.Incr(key).Result()
        m.redis.Expire(key, time.Minute)

        if count > 100 { // 每分钟最多100事件
            c.JSON(429, gin.H{"error": "Too many requests"})
            c.Abort()
            return
        }
        c.Next()
    }
}
```
